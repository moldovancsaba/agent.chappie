## Mandatory handover discipline

Every handover must include:

- what documentation was used during implementation
- what assumptions were explicitly avoided
- what ambiguities were clarified (or escalated)
- proof artifacts (tests, API request/response, DB evidence when relevant)
- remaining unknowns and the exact question for the Architect

## Agent Work Rule

When you are done with a bucket of tasks, always provide a full plain text summary that can be easily copied and pasted for the Architect to review.
