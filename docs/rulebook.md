# Rulebook

## Core execution discipline

- Do not assume undocumented behavior.
- Use repository documentation as active operating memory.
- Clarify ambiguity before acting on critical changes.
- Keep code, contracts, runbooks, and handoffs aligned.
- Treat deterministic proof as part of implementation, not optional follow-up.

## Clarify-before-act rule

If required behavior is not explicit in docs + code, stop and ask before implementing.

## Agent Work Rule
When you are done with a bucket of tasks, always provide a full plain text summary that can be easily copied and pasted for the Architect to review.
